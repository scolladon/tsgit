# ADR-093: Drop `DESIGN.md` and `MIGRATION.md` without redirect stubs

## Status

Proposed.

## Context

The audience-first restructure (ADR-092) absorbs both root-level documents into the new funnels:

- `DESIGN.md` → `docs/understand/{architecture,security,performance}.md`
- `MIGRATION.md` → `docs/get-started/migrate-from-isomorphic-git.md`

The question is what to do with the original files. Three options:

1. **Delete outright** — `git rm`.
2. **Keep a 2-line redirect stub** — file stays, body says "moved, see X".
3. **Move the redirect text into `CLAUDE.md`** — relies on agent / contributor context.

A grep audit established the inbound link surface for each file:

| File | Inbound references |
|---|---|
| `DESIGN.md` | `CLAUDE.md` step 8 (workflow), `CONTRIBUTING.md` step "Update docs", 8 historical `docs/plan/*.md` snapshots, `reports/api/media/CONTRIBUTING.md` (TypeDoc-generated copy). No `README.md` link. No `package.json` link. No external mention discovered. |
| `MIGRATION.md` | `README.md` §Compatibility shim only (removed by the README rewrite), `CHANGELOG.md` (historical), 4 historical `docs/plan/*.md`. No `package.json` link. No npm-surfaced link. |

Both files are referenced **only inside the repository**. npm's published `homepage` field points at `https://github.com/scolladon/tsgit#readme`, never at these files. The README has never linked to `DESIGN.md`.

## Decision

**Delete both files outright.** No redirect stubs.

Updates that accompany the deletion (single PR, same commit chain):

- `CLAUDE.md` step 8 — replace `DESIGN.md` with `docs/understand/*`.
- `CONTRIBUTING.md` step "Update docs" — same swap.
- `README.md` — references to `MIGRATION.md` already absent in the rewritten version.
- Historical `docs/plan/*.md` — **left untouched**. They are snapshots of decisions taken at a moment; rewriting them lies about what the plan said at the time.
- `reports/api/media/CONTRIBUTING.md` — regenerated by the next TypeDoc run; no manual edit.

`CLAUDE.md` is **not** used as the landing page for redirect text. CLAUDE.md serves the AI agent during sessions; a human clicking a stale GitHub URL never sees it.

## Consequences

### Positive

- Two root-level files removed; the repo root is leaner.
- No "redirect debt" — no 2-line files that need maintenance whenever the destination paths change.
- Workflow rules in `CLAUDE.md` / `CONTRIBUTING.md` now point at the active documentation surface; no indirection.
- Future grep for `DESIGN.md` / `MIGRATION.md` returns nothing relevant — the absence is unambiguous.

### Negative

- A bookmark to `https://github.com/scolladon/tsgit/blob/main/DESIGN.md` will 404. The grep audit found no external references, but unknown personal bookmarks are unobservable. GitHub serves `404` with a "this file does not exist" page that mentions recent commits — the deletion commit is one of them, and its message lands in the GitHub search result.
- Anyone with a forked branch that still references the file from a workflow doc will hit a stale reference on rebase. Mitigation: the deletion commit message explicitly names the new locations, and the moves are mechanical.

### Neutral

- The deletion is reversible — `git revert` restores both files. ADR-093 itself would then be revisited and superseded.
- The historical `docs/plan/*.md` references to `DESIGN.md` continue to exist. They describe a state of the world that was true at their commit; leaving them is more honest than rewriting history. Future contributors reading old plans encounter a file that no longer exists and can resolve via this ADR or git history.

## Alternatives considered

- **Keep 2-line redirect stubs** — rejected because the grep audit found no external link discovery. Stubs would carry maintenance cost (every time the destination folder restructures) with no measurable benefit.
- **Move the redirect text into `CLAUDE.md`** — rejected because `CLAUDE.md` is read by the AI agent, not by human visitors clicking a stale `blob/main/DESIGN.md` URL. Wrong audience.
- **Rewrite the historical `docs/plan/*.md` files to point at the new paths** — rejected because those files are snapshots of decisions made at their commit. Mutating them retroactively lies about what the plan said. The current accepted convention (per `CLAUDE.md`: "No phase refs in code") already accepts that historical artifacts reference filenames that may have moved.
- **Soft-delete via `.gitattributes` `export-ignore`** — rejected; the files would still exist on `main`, so the redirect-vs-delete trade-off is unchanged.
